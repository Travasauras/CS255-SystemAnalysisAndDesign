For this project, I was tasked with designing the a website for DriverPass to allow students to prep for getting their license from the DMV.
The client wanted a functional system where users could login, practice DMV tests, schedule driving lessons with instructors, and check their
progress to ensure they were ready for the real DMV exam.  They wanted administrative access as well as realtime updates from the DMV. 

I believe I captured the use cases as well as the class diagram pretty well.  These diagrams can be very useful when splitting a project
up between a development team, allowing them to have a good idea how the system should work as a whole while only working on a single part
of the project. 

If I could revise part of this project, it would probably be the security aspect of the business requirements.  While I believe I covered
the basic security needs, I would like to add more security like a 2factor authentication and captcha check to weed out bots.

After reading the interview document, I took into consideration what Liam asked for and designed a system that could implement it in a way
that met all of his requests. I added funcitonality and security in places that didn't interfere with his requests, which added value 
to the system iteslf. 

When I am designing software, I like to think about the end users and how they may interact with the system once it is finished.  Ultimately
the client gets the final say in what the product is, but if the user has a hard time with the product, then it will not be recieved well.
UML class diagrams as well as flowcharts are two strategies I can see myself continuing to use in the future when designing systems.
